Tutorials

Code Examples

Hands-on tutorials to get you building with CURS3D. From your first node to Docker networks and REST API integration.

Tutorial 1

Run Your First Node

Install CURS3D from source and start a blockchain node on your local machine in under five minutes.

1 Install Rust

terminal
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
source $HOME/.cargo/env

2 Clone and Build

terminal
git clone https://github.com/Pazificateur69/curs3d.git
cd curs3d
cargo build --release

3 Create a Wallet

terminal
./target/release/curs3d wallet --output my_wallet.json

4 Start the Node

Run a relay node. The node initializes the chain, starts the TCP RPC on port 9545 and the HTTP REST API on port 8080.

terminal
./target/release/curs3d node --port 4337 --data-dir node1_data

Verify the node is running:

terminal (new tab)
curl http://localhost:8080/api/status
Tutorial 2

Send Your First Transaction

Create two wallets, fund one via genesis allocation, and send tokens between them.

1 Create Two Wallets

terminal
./target/release/curs3d wallet --output alice.json
./target/release/curs3d wallet --output bob.json

2 Create Genesis Config

Get Alice's public key from curs3d info --wallet alice.json and create a genesis.json:

genesis.json
{
  "chain_name": "tutorial-net",
  "block_reward": 50000000,
  "minimum_stake": 1000000000,
  "allocations": [
    {
      "public_key": "PASTE_ALICE_HEX_PUBLIC_KEY_HERE",
      "balance": 10000000000,
      "staked_balance": 2000000000
    }
  ]
}

3 Start a Validator Node

terminal
./target/release/curs3d node \
  --port 4337 \
  --data-dir tutorial_data \
  --validator-wallet alice.json \
  --genesis-config genesis.json

4 Send Tokens to Bob

terminal (new tab)
./target/release/curs3d send \
  --wallet alice.json \
  --to BOB_ADDRESS_HERE \
  --amount 500 \
  --fee 1000

5 Verify via REST API

Check chain status and look up Bob's balance using the HTTP API:

terminal
# Check chain status
curl http://localhost:8080/api/status

# Check Bob's balance (use Bob's address)
curl http://localhost:8080/api/account/BOB_ADDRESS_HERE
Tutorial 3

Become a Validator

Stake tokens to register your node as a block-producing validator on an existing network.

1 Submit Stake Transaction

terminal
./target/release/curs3d stake \
  --wallet validator.json \
  --amount 1000 \
  --fee 1000

2 Run as Validator

terminal
./target/release/curs3d node \
  --port 4337 \
  --data-dir curs3d_data \
  --validator-wallet validator.json

3 Verify Validator Status

terminal
# Check active validators via REST API
curl http://localhost:8080/api/validators

# Check chain status with finalized height
./target/release/curs3d status --rpc-addr 127.0.0.1:9545
Tutorial 4

Multi-Node Network

Connect multiple nodes on the same machine and watch them sync blocks via Gossipsub.

1 Start Node A (Validator)

terminal 1
./target/release/curs3d node \
  --port 4337 \
  --data-dir node_a_data \
  --validator-wallet alice.json \
  --genesis-config genesis.json \
  --rpc-addr 127.0.0.1:9545

2 Start Node B (Relay)

terminal 2
./target/release/curs3d node \
  --port 4338 \
  --data-dir node_b_data \
  --genesis-config genesis.json \
  --rpc-addr 127.0.0.1:9546 \
  --bootnode /ip4/127.0.0.1/tcp/4337

3 Verify Both Nodes

terminal
# Both should report the same height and latest hash
curl http://localhost:8080/api/status
curl http://localhost:8081/api/status
Tutorial 5

Use the REST API

Interact with a running CURS3D node via HTTP. All 10 endpoints with curl examples.

1 Chain Status

terminal
curl http://localhost:8080/api/status | python3 -m json.tool

2 Browse Blocks

terminal
# Latest 20 blocks
curl "http://localhost:8080/api/blocks?from=0&limit=20"

# Specific block detail
curl http://localhost:8080/api/block/5

3 Account Lookup

terminal
curl http://localhost:8080/api/account/CUR7f3a8b1c2d4e5f6a7b8c9d0e1f2a3b4c5d6e7f8

4 Validators and Faucet

terminal
# List active validators
curl http://localhost:8080/api/validators

# Use testnet faucet (100 CUR)
curl http://localhost:8080/api/faucet/YOUR_ADDRESS_HERE

5 Transaction Search and Pending

terminal
# Search transaction by hash
curl http://localhost:8080/api/tx/a1b2c3d4...

# View pending transactions in mempool
curl http://localhost:8080/api/pending
Tutorial 6

Unstake Your Tokens

Unstake moves tokens from your staked balance back to liquid balance. If your remaining stake drops below the minimum, you will be deactivated as a validator.

1 Check Current Stake

terminal
curl http://localhost:8080/api/account/YOUR_ADDRESS
# Look for "staked_balance" in the response

2 Submit Unstake via API

Unstake transactions can be submitted via the REST API. The transaction kind is Unstake with the amount to unstake:

terminal
curl -X POST http://localhost:8080/api/tx/submit \
  -H "Content-Type: application/json" \
  -d '{"kind":"Unstake","from":[...],"sender_public_key":[...],"to":[],"amount":500000000,"fee":1000,"nonce":1,"timestamp":1234567890,"signature":{"bytes":[...]}}'

3 Verify

After the unstake transaction is included in a block, your liquid balance increases and your staked balance decreases.

terminal
curl http://localhost:8080/api/account/YOUR_ADDRESS
Tutorial 7

Run with Docker

Launch a 2-node CURS3D network with a single docker-compose command.

1 Build and Launch

terminal
git clone https://github.com/Pazificateur69/curs3d.git
cd curs3d
docker-compose up -d

2 Check Status

terminal
# Node 1
curl http://localhost:8080/api/status

# Node 2
curl http://localhost:8081/api/status

# View logs
docker-compose logs -f

3 Clean Up

terminal
docker-compose down
Tutorial 8

Use the Block Explorer

The built-in Block Explorer connects to the REST API and provides a visual interface for browsing the chain.

1 Start a Node

Start any CURS3D node -- the HTTP REST API must be running on port 8080.

2 Open the Explorer

Open explorer.html in your browser. The default API endpoint is http://localhost:8080. You can change it in the config bar at the top.

3 Explore the Chain

The explorer shows:

  • Chain height, finalized height, active validators, pending txs
  • Recent blocks table -- click a row to expand and see transactions
  • Account lookup by address (balance, nonce, staked balance)
  • Transaction search by hash
  • Testnet faucet -- request 100 CUR to any address
Tutorial 9

Custom Genesis Configuration

Create a genesis file that pre-funds multiple accounts for a private testnet.

1 Generate Wallets and Get Public Keys

terminal
./target/release/curs3d wallet --output validator1.json
./target/release/curs3d wallet --output validator2.json
./target/release/curs3d info --wallet validator1.json
./target/release/curs3d info --wallet validator2.json

2 Write Genesis File

genesis.json
{
  "chain_name": "my-private-testnet",
  "block_reward": 25000000,
  "minimum_stake": 500000000,
  "allocations": [
    {
      "public_key": "VALIDATOR_1_PUBLIC_KEY_HEX",
      "balance": 1000000000,
      "staked_balance": 2000000000
    },
    {
      "public_key": "VALIDATOR_2_PUBLIC_KEY_HEX",
      "balance": 1000000000,
      "staked_balance": 2000000000
    }
  ]
}

3 Launch the Network

terminal
# Terminal 1
./target/release/curs3d node --port 4337 --data-dir v1_data \
  --validator-wallet validator1.json --genesis-config genesis.json

# Terminal 2
./target/release/curs3d node --port 4338 --data-dir v2_data \
  --validator-wallet validator2.json --genesis-config genesis.json \
  --bootnode /ip4/127.0.0.1/tcp/4337
