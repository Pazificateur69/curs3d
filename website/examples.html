Examples

Learn CURS3D by actually using it.

These examples are designed to match the current protocol surface: node bootstrap, validator flow, faucet requests, API inspection and state-oriented queries.

1Bootstrap a local node

Build the binary, create a wallet, generate a genesis file, and run a local validator node.

terminal
git clone https://github.com/Pazificateur69/curs3d.git
cd curs3d
cargo build --release

printf '%s\n' 'change-this-validator-password' > validator.password
./target/release/curs3d wallet --output validator.json --password-file validator.password

./target/release/curs3d genesis \
  --output genesis.public-testnet.json \
  --validator-wallet validator.json \
  --validator-password-file validator.password

./target/release/curs3d node \
  --validator-wallet validator.json \
  --validator-password-file validator.password \
  --genesis-config genesis.public-testnet.json

2Publish a stable bootnode address

For a public testnet or VPS deployment, publish a public address tied to the node identity stored in the data directory.

terminal
./target/release/curs3d bootnode-address \
  --data-dir curs3d_data \
  --public-addr /dns4/node.example.com/tcp/4337

3Inspect chain status and validators

Use the HTTP API to verify the chain height, epoch and active validator set.

terminal
curl http://127.0.0.1:8080/api/status | jq .data
curl http://127.0.0.1:8080/api/validators | jq .data
curl http://127.0.0.1:8080/api/metrics

4Request faucet funds with JSON POST

The faucet is intentionally exposed as a POST endpoint with a JSON body, not as a public GET side effect.

terminal
curl -X POST http://127.0.0.1:8080/api/faucet/request \
  -H 'Content-Type: application/json' \
  -d '{"address":"CUR0123456789abcdef0123456789abcdef01234567"}'

5Send value and inspect account state

Submit a transfer through the CLI, then inspect the receiver with the account endpoint.

terminal
./target/release/curs3d send \
  --wallet validator.json \
  --password-file validator.password \
  --to CUR0123456789abcdef0123456789abcdef01234567 \
  --amount 100 \
  --fee 1000

curl http://127.0.0.1:8080/api/account/CUR0123456789abcdef0123456789abcdef01234567 | jq .data

6Inspect token and governance state

Once a network has token and governance activity, these read endpoints surface the current public state.

terminal
curl http://127.0.0.1:8080/api/tokens | jq .data
curl http://127.0.0.1:8080/api/governance/proposals | jq .data
curl http://127.0.0.1:8080/api/pending | jq .data
