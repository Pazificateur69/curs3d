Core Features

Engineered for the post-quantum era

Every component built from scratch to resist quantum attacks while delivering production-grade blockchain infrastructure with smart contract execution.

◊

Post-Quantum Cryptography

CRYSTALS-Dilithium Level 5 (NIST FIPS 204) for all signatures. SHA-3 Keccak-256 with double-hash blocks and Merkle trees. AES-256-GCM + Argon2 wallet encryption.

◇

WASM Smart Contracts

Deterministic WebAssembly VM powered by Wasmer with Cranelift. Deploy and call contracts with per-instruction gas metering, key-value storage, and execution receipts.

≡

BFT Finality

Byzantine Fault Tolerant Proof of Stake with 2/3 stake-weighted threshold finality. Epoch-based validator sets frozen per epoch for deterministic proposer selection.

◆

EIP-1559 Fee Market

Dynamic base fee per gas with configurable change denominator. Separate max_fee and priority_fee per gas. Automatic gas refunds on unused gas. Mempool priority ordering.

⊥

Fork Choice Rule

BlockTree with heaviest-chain rule by cumulative proposer stake. Automatic reorg detection with finality boundary. Non-canonical branch pruning.

×

Provable Slashing

Cryptographic EquivocationEvidence with dual Dilithium signatures proves double-signing. 33% stake penalty and configurable jail duration (default 64 blocks).

⇄

State Sync

Full state snapshots with Merkle-verified chunks. Network protocol for snapshot manifest and chunk transfer. Fast sync from finalized checkpoints.

↑

Protocol Upgrades

Versioned protocol with upgrade-at-height activation. Network topic derived from chain_id and protocol version. Automatic peer compatibility filtering.

counter.wat 
;; A simple counter contract for CURS3D
;; Uses storage_get / storage_set host functions

(module
  (import "curs3d" "storage_get"
    (func $storage_get (param i64) (result i64)))
  (import "curs3d" "storage_set"
    (func $storage_set (param i64 i64)))
  (import "curs3d" "emit_log"
    (func $emit_log (param i64 i64)))
  (import "curs3d" "consume_gas"
    (func $consume_gas (param i64)))

  (memory (export "memory") 1)

  (func (export "curs3d_call") (result i64)
    ;; Read current counter value
    (local $count i64)
    (local.set $count
      (call $storage_get (i64.const 0)))
    ;; Increment and store
    (local.set $count
      (i64.add (local.get $count) (i64.const 1)))
    (call $storage_set
      (i64.const 0) (local.get $count))
    ;; Emit event
    (call $emit_log
      (i64.const 1) (local.get $count))
    ;; Return new value
    (local.get $count))
)

crypto/dilithium.rs 
use pqcrypto_dilithium::dilithium5::{
    DetachedSignature, PublicKey, SecretKey,
    detached_sign, keypair, verify_detached_signature,
};

pub struct KeyPair {
    pub public_key: Vec<u8>,
    pub secret_key: Vec<u8>,
}

impl KeyPair {
    pub fn generate() -> Self {
        let (pk, sk) = keypair();
        KeyPair {
            public_key: pk.as_bytes().to_vec(),
            secret_key: sk.as_bytes().to_vec(),
        }
    }

    pub fn sign(&self, message: &[u8]) -> Signature {
        let sk = SecretKey::from_bytes(&self.secret_key)
            .expect("invalid secret key");
        let sig = detached_sign(message, &sk);
        Signature(sig.as_bytes().to_vec())
    }
}

pub fn verify(
    message: &[u8],
    signature: &Signature,
    public_key: &[u8],
) -> bool {
    let pk = PublicKey::from_bytes(public_key);
    let sig = DetachedSignature::from_bytes(&signature.0);
    verify_detached_signature(&sig, message, &pk)
        .is_ok()
}
