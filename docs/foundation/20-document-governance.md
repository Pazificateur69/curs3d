# Document Governance

## Pourquoi

Un projet perd vite en crédibilité quand sa documentation diverge entre:
- repo
- site
- deck
- docs internes
- messages publics

## Règle

Toute affirmation importante sur `CURS3D` doit avoir une source officielle et un propriétaire.

## À versionner explicitement

- slogan principal
- positioning statement
- thesis statement
- description produit courte
- description investisseur courte
- nombre de types de transactions
- routes API publiques
- état du testnet et du mainnet

## Responsabilités

- fondateur: vision, thèse, priorités
- engineering: exactitude technique
- branding/communication: cohérence de forme
- fundraising: cohérence deck et data room

## Cadence recommandée

- revue rapide hebdomadaire des changements visibles
- revue mensuelle des docs de fond
- revue obligatoire avant toute annonce publique importante

## Principe de crédibilité

Mieux vaut une doc plus courte mais juste qu'une doc abondante mais contradictoire.
