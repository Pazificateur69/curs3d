# Mainnet Credibility Checklist

## Technique

- `cargo test` propre
- `clippy -D warnings` propre
- testnet stable
- audit externe
- bug bounty
- observabilité
- procédures de release

## Produit

- docs propres
- wallet
- explorer
- onboarding dev
- endpoints stables

## Marché

- partenaires pilotes
- cas d'usage réels
- thèse répétable
- assets de marque cohérents

## Gouvernance

- structure claire
- processus d'upgrade
- gestion incidents

## Investisseur

- deck
- one-pager
- tokenomics
- roadmap
- data room

## Règle simple

Si un visiteur sérieux ne comprend pas:
- pourquoi le projet existe
- pourquoi maintenant
- pourquoi il est différent
- pourquoi il peut survivre

alors la crédibilité n'est pas encore suffisante.
