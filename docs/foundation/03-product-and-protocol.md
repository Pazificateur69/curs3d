# Product And Protocol

## Ce qu'est le produit

`CURS3D` doit être vu à trois niveaux:

### Niveau 1: protocole

- consensus
- crypto
- exécution
- état
- gouvernance

### Niveau 2: plateforme

- node
- wallet
- explorer
- API/RPC
- docs
- outils launch / token / gouvernance

### Niveau 3: écosystème

- validateurs
- projets
- partenaires
- infrastructure tierce

## Proposition produit

La proposition produit n'est pas seulement:

`voici une blockchain`

Elle doit devenir:

`voici la meilleure base pour émettre et opérer des actifs conçus pour durer`

## Éléments indispensables de crédibilité produit

- testnet stable
- docs de déploiement
- wallet utilisable
- explorer propre
- endpoints clairs
- démonstration de token issuance
- démonstration de gouvernance
- feuille de route mainnet réaliste

## Différenciation protocolaire

`CURS3D` doit défendre:
- crypto post-quantique native
- langage clair sur la durabilité
- gouvernance et upgrades réfléchies
- orientation vers des actifs trust-critical

## Piège à éviter

Ne pas empiler des features sans thèse produit.

Chaque composant doit renforcer la promesse centrale:
- sécurité long terme
- actifs durables
- trust layer crédible
