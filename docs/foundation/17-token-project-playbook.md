# Token Project Playbook

## Objectif

Donner aux projets tiers une raison claire de lancer un token ou une application sur `CURS3D`.

## Promesse partenaire

`CURS3D` doit offrir:
- une histoire de sécurité long terme
- une marque différenciante
- un support d'intégration propre
- une couche de confiance cohérente avec des actifs conçus pour durer

## Cibles prioritaires

- projets token premium
- infrastructures orientées conformité et traçabilité
- applications qui veulent une image future-proof
- projets avec horizon de confiance long

## Ce qu'un partenaire veut voir

- docs simples
- endpoints clairs
- support humain
- calendrier réaliste
- visibilité sur la roadmap
- crédibilité opérationnelle

## Assets à préparer pour eux

- one-pager partenaire
- FAQ intégration
- checklist de lancement token
- guide branding "built on CURS3D"
- contacts techniques

## Arguments utiles

- différenciation narrative immédiate
- infrastructure plus lisible pour actifs durables
- proximité avec l'équipe fondatrice au stade initial
- possibilité de devenir cas d'usage vitrine du réseau

## Erreurs à éviter

- promettre une liquidité inexistante
- exagérer l'écosystème
- vendre la chaîne comme universelle
- négliger l'onboarding technique
